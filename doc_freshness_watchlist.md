# Doc Freshness Watchlist

Relative path: `doc_freshness_watchlist.md`

Purpose: This document lists the repo-authored docs that should be reviewed for freshness when work changes the relay's behavior, process, planning state, or maintenance model. Use it during checkpoint and milestone maintenance to avoid stale documentation and keep the system coherent across roadmap, backlog, history, architecture, and ritual assets.

Refresh these docs whenever a checkpoint or milestone touches their subject area.

| Doc | Refresh trigger |
| --- | --- |
| `README.md` | User-facing behavior, setup, or repo purpose changes |
| `docs/architecture/master_architecture_map.md` | Orchestrator flow or major state transitions change |
| `docs/architecture/system_module_map.md` | Modules, seams, or ownership boundaries change |
| `docs/architecture/pipeline_contract_sheet.md` | Artifact shapes, relay workflow outputs, or relay I/O expectations change |
| `docs/process/development_playbook.md` | Ritual behavior or operating defaults change |
| `docs/process/relay_workflow_operating_model.md` | Slice/checkpoint/milestone workflow rules change |
| `docs/process/ritual_contracts.md` | Wrapper inputs/outputs change |
| `docs/process/relay_version_journal.md` | New checkpoint or milestone is completed |
| `docs/process/roadmap_to_mvp.md` | Near-term priorities or current MVP phase changes |
| `docs/process/backlog.md` | Future phases, wish list items, or discussion-needed ideas change |
| `docs/process/CHANGELOG.md` | Work is implemented and verified |
| `docs/process/developer_journal.md` | Checkpoint or milestone narrative should be recorded |
| `docs/process/docs_system_wiring_guide.md` | The wiring between docs, briefs, rituals, and skills changes |
