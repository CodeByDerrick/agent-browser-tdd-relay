# Active Brief Lane

Relative path: `automation/briefs/active/README.md`

Purpose: This document explains what belongs in the active brief lane. Use it to keep only the current or next-immediate brief in the active area so slice work stays low-noise and token-efficient. It works with the brief template, archive lane, roadmap, and ritual system by keeping only actionable brief material close to execution.

Place only the current working brief or the next immediately approved brief here.
