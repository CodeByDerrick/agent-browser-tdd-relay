# Doc Update Checklist

Relative path: `automation/checklists/doc-update-checklist.md`

Purpose: This checklist defines the baseline expectations for editing repo-authored docs in `agent-browser-tdd-relay`. Use it to keep documentation compact, consistent, self-describing, and aligned with the relay workflow whenever a doc refresh happens.

- Update only docs touched by the slice/checkpoint/milestone
- Prefer compact relay-specific language
- Preserve relay workflow rules and brief contract language
- Preserve the title/path/purpose header standard
- Keep examples short and token-efficient
