# Slice Ritual Checklist

Relative path: `automation/checklists/slice-ritual-checklist.md`

Purpose: This checklist defines the operational checks for running the default slice ritual in `agent-browser-tdd-relay`. Use it to confirm that a brief is small enough, the TDD flow is explicit, and the slice output stays aligned with the relay workflow.

- Confirm the brief is a minimal vertical slice
- Confirm the brief states red/green intent
- Confirm verification and refactor expectations are clear enough for the slice
- Confirm touched seam(s) are named
- Run TDD from the brief with minimal context
- Capture slice evidence and report
- Produce the next brief candidate in minimal form
- If this is the 3rd approved slice, hand off to checkpoint ritual
