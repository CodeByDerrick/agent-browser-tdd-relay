# Checkpoint Ritual Checklist

Relative path: `automation/checklists/checkpoint-ritual-checklist.md`

Purpose: This checklist defines the recurring maintenance work that should happen at the checkpoint boundary in `agent-browser-tdd-relay`. Use it after every 3 approved slices to keep docs, planning state, repo maintenance, and historical records aligned before the next slice batch begins.

- Confirm 3 approved slices have completed
- Gather the last slice reports and current repo state
- Refresh affected docs from the watchlist
- Reconcile repo drift and process drift
- Review roadmap and backlog drift
- Produce a checkpoint packet
- Prepare commit recommendation
- Record relay version journal entry
- Update developer journal if checkpoint narrative matters
- Recommend checkpoint git tag/label
- Author the next minimal brief
