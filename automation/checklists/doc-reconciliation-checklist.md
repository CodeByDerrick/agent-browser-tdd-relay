# Doc Reconciliation Checklist

Relative path: `automation/checklists/doc-reconciliation-checklist.md`
Project: Agent Bridge

Purpose: This checklist defines how to reconcile code and documentation when drift is possible in Agent Bridge. Use it during checkpoint or milestone maintenance to verify that process docs, architecture docs, and planning or history docs still match the current repo state.

- Compare changed code against affected docs
- Update docs that describe changed behavior or boundaries
- Note any intentional stale docs in the ritual packet
- Confirm no wrapper references orphaned docs
- Confirm roadmap, backlog, and history docs still reflect the current state where relevant
