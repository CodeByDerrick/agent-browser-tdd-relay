# Doc Reconciliation Checklist

Relative path: `automation/checklists/doc-reconciliation-checklist.md`

Purpose: This checklist defines how to reconcile code and documentation when drift is possible in `agent-browser-tdd-relay`. Use it during checkpoint or milestone maintenance to verify that process docs, architecture docs, and planning/history docs still match the current repo state.

- Compare changed code against affected docs
- Update docs that describe changed behavior or boundaries
- Note any intentional stale docs in the ritual packet
- Confirm no wrapper references orphaned docs
- Confirm roadmap/backlog/history docs still reflect the current state where relevant
